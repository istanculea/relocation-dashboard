# Security Policy

## Supported versions

This repository is in active development. Security fixes are applied on the default branch.

## Reporting a vulnerability

Please do **not** open a public issue for suspected vulnerabilities.

Instead, report security issues privately to the repository owner via GitHub security reporting or direct maintainer contact.

## Secure development baseline

- Never commit secrets or credentials. Use environment variables for sensitive values.
- Keep dependencies up to date and patch known vulnerabilities promptly.
- Validate and sanitize all user-provided input before using it.

# UX Wireframe Map: Four Primary Modes

## Global Shell

## Structure
- Top Context Bar
- Left Strategic Rail
- Main Intelligence Canvas
- Right Insight Rail

## Top Context Bar
- Active subject scope (region, corridor, city).
- Lens selector.
- Horizon selector.
- Simulation state chip.
- Search field.
- Share and export actions.

## Left Strategic Rail
- Map layers and compositing controls.
- Scenario toggles.
- Lens presets.
- Temporal controls.
- Candidate set actions.

## Right Insight Rail
- Strategic interpretation.
- Tradeoff narrative.
- Confidence gradient summary.
- Evidence entry points.

## Primary Mode 1: Strategic Atlas

## Layout
- Canvas dominated by full-screen map.
- Left rail for overlays and mobility filters.
- Right rail for context and confidence.
- Optional bottom timeline strip.

## Wireframe Blocks
1. Atlas Header Strip
- Scope and horizon.
- Selected corridor chip.

2. Layer Stack Control
- Rail corridors
- Airport influence
- Healthcare reachability
- Affordability pressure
- Climate comfort
- Confidence atmosphere

3. Mobility Halo Surface
- Reachable capitals
- Reachable GDP
- Reachable population
- Network redundancy

4. Context Rail Cards
- Why this corridor is strategically strong
- Key vulnerabilities
- Confidence and freshness summary

## Interaction Path
1. Select corridor or region.
2. Add up to 3 composited layers.
3. Hover city node to preview summary.
4. Pin subject to compare workspace.

## Primary Mode 2: Compare Workspace

## Layout
- Main canvas split into three vertical zones:
  - Narrative summary strip (top)
  - Delta visuals and trajectories (middle)
  - Confidence matrix and evidence (bottom)

## Wireframe Blocks
1. Selection Dock
- Up to 8 entities (cities, regions, clusters, corridors).
- Lens and horizon control.

2. Strategic Summary
- Ranked strengths by scenario relevance.
- Tradeoff list with plain-language rationale.

3. Divergence Panel
- Metric slope deltas.
- Trajectory divergence markers.

4. Family and Mobility Panels
- Family rhythm compatibility map.
- Mobility positioning and resilience map.

5. Confidence Matrix
- Per metric confidence class.
- Source diversity and freshness status.

## Interaction Path
1. Build candidate set.
2. Select lens.
3. Compare narrative and deltas.
4. Expand any delta into evidence chain.

## Primary Mode 3: Future Outlook

## Layout
- Left: temporal map and replay timeline.
- Right: projection stack with uncertainty bands.
- Bottom: horizon scrubber and scenario chips.

## Wireframe Blocks
1. Timeline Controller
- Current, 5-year, 10-year, long horizon.
- Play and pause temporal replay.

2. Projection Panels
- Affordability trend
- Climate stress trend
- Mobility evolution
- Infrastructure momentum
- Demographic and governance pressure

3. Volatility Surface
- Confidence trend (improving, stable, deteriorating).
- Volatility markers per metric family.

4. Scenario Delta Card
- Baseline vs shock scenario deltas.

## Interaction Path
1. Select horizon.
2. Activate scenario.
3. Replay projection.
4. Capture snapshot into candidate set.

## Primary Mode 4: Family Rhythm Explorer

## Layout
- Softer map layers with rhythm overlays.
- Right rail emphasizes practical interpretation over raw metrics.

## Wireframe Blocks
1. Household and Persona Setup
- Household composition
- Mobility dependence
- Sensory profile

2. Rhythm Overlay Stack
- Calmness gradient
- Noise and sensory load
- Child autonomy zones
- Bureaucracy friction
- Green recovery access

3. Human Interpretation Rail
- Stress exposure narrative.
- Daily logistics realism.
- Confidence-aware caveats.

4. Candidate Notes Board
- Annotated rationale.
- Mode-specific snapshot comments.

## Interaction Path
1. Configure household profile.
2. Explore rhythm overlays.
3. Read interpretation and caveats.
4. Save strategic candidate notes.

## Accessibility and Reduced Complexity Views

1. Reduced motion profile
- No inertial transitions.
- Minimal animation.

2. Alternate non-map mode
- Narrative-first table and chart fallback.

3. Keyboard and screen reader support
- Full focus order.
- Landmark roles per shell region.
- Described confidence layers.
